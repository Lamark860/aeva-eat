### Приоритет 0 — критический путь (до показа кругу)

- [ ] **Cover-fallback fix** — `COALESCE(p.image_url, первое review.image_url)` не отрабатывает, 80% полароидов пустые (`R6 §A1–A2`)
- [ ] **Активация `isTicketOnly`** — условие `!!place.image_url` пропускает пустые строки; заменить на `!!(place.image_url || '').trim()` (`R6 §A3`)

### Приоритет 1 — обязательно до показа

- [ ] Штамп-помойка в шапке места — максимум 2 штампа (город + кухня) + жемчужина (`R6 §B3`)
- [ ] R5-Q1: убрать гендерный глагол, формат `«жемчужина · Аня · 12 марта»` (`R6 §B4`)
- [ ] «Любит X» — показывать только если `≥5 визитов И ≥15% от общего числа` (`R6 §B7`)
- [ ] Билетик в профиле — починить ячейку «городов» (прочерк вместо числа) (`R6 §B7`)
- [ ] Маркер карты — проверить почему ромб показывает 3.5 при общем рейтинге 8.8 (`R6 §B6`)

### Приоритет 2 — следующая итерация

- [ ] Public share: пропорции (cover 40–50%, CTA на первом экране) + `?next=` для R5-Q3 (`R6 §B5`)
- [ ] Сидинг разнообразного fake-набора (25–30 мест: с фото / без / с видео / gem / wishlist / записки)
- [ ] Город — переоборудовать в путеводитель (curated полароиды + мини-карта + цитата) (`R6 §B1`)
- [ ] Жемчужины — переоборудовать в hub сокровищ, не таблицу (`R6 §B2`)
- [ ] Person page — reduction-strategy для длинных профилей (`R6 §B8`)
- [ ] R5-Q4: tilt 1–2° на wishlist-карточках
- [ ] R5-Q5: «тапни, чтобы посмотреть» после 500мс
- [ ] R5-Q6: разделитель-плашка «… а вот эти Аня ещё не пробовала» при `sort=rating_user`

### Структурные (рефакторинг)

- [ ] Разнести `ArtifactCard.vue` на `ArtifactPolaroid` / `ArtifactTicket` / `KruzhokStack` + тонкий роутер сверху (`R6 §D1`)
- [ ] Правило приоритета full-width: `gem > video > первый place`, max 1 на неделю; `ticket-only` тоже может быть featured (`R6 §D2`)
- [ ] Убрать `grid-auto-flow: dense` или ввести композицию «полароид 2/3 + маленький 1/3» (`R6 §D3`)
- [ ] Лог `kruzhok-miss` для измерения промахов hit-target (`R6 §D4`)
- [ ] Убрать `.onboarding-cta` снизу — оставить только баннер сверху (`R6 §D5`)

## Запланировано (next)

### Бэк-фундамент без UI — нужно дотянуть UI

- [ ] Кнопка «по оценке N» в шапке места (бэк уже отдаёт `ratings_per_user`)
- [ ] Полная интеграция `attended_by` / `visit_from-to` в UI фильтров

### Инфраструктура

- [ ] CI/CD (GitHub Actions: линт + тесты + docker build)
- [ ] Продакшн-деплой на VPS (есть `docker-compose.prod.yml`, не прокачано)
- [ ] Бэкап БД по расписанию (cron + pg_dump → S3/local)
- [ ] pgAdmin в `docker-compose.yml` (веб-интерфейс БД для разработки)
- [ ] Rate limiting на API (защита от спама)
- [ ] ffmpeg-генерация постеров для видео-кружков (решает R5-Q5 на iOS)
- [ ] HTTPS-туннель для записи видео через WiFi (Tailscale)

### Админка

- [ ] Панель superuser: CRUD кухонь/категорий/тегов из UI
- [ ] Управление пользователями: блокировка, смена роли (API готов — `/api/admin/users`, нет UI)
- [ ] Модерация отзывов: просмотр всех, удаление неуместных
- [ ] Дашборд: кол-во заведений/отзывов/юзеров, графики

### Заведения

- [ ] Статус заведения: открыто / временно закрыто / закрылось навсегда
- [ ] Часы работы (расписание по дням)
- [ ] Средний чек (диапазон цен)
- [ ] Ссылка на меню / фото меню
- [ ] Контакты (телефон, Instagram, Telegram)
- [ ] Пользовательские теги (например: «есть веранда», «подходит для свиданий», «шумно»)

### Отзывы

- [ ] Комментарии к отзывам (обсуждение с друзьями)
- [ ] Reactions (C5) — `жму руку` / `тоже хочу` / `там был` штампики (после MVP)

### UI/UX

- [ ] Тёмная тема (G2) — токены подготовлены, фактической темы нет (отложено в next-next, ~3 мес после запуска)
- [ ] Page-transitions между маршрутами
- [ ] Свайпы (мобильные)
- [ ] «Ваш год / сезон» (C4) — раз в полгода открытка-итог (через ~3 мес после запуска)

### Качество

- [ ] Фронтенд-тесты (vitest / Playwright) — сейчас 0 тестов
- [ ] 5 eslint-warnings (`LocationPicker.vue`, `PlaceDetail.vue`)
- [ ] `components/LocationPicker.vue` — единственный неперевёрстанный Bootstrap-компонент
