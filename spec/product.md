# AEVA Eat — продукт

## Философия

Камерное приватное приложение для круга друзей (3-30 человек). Два режима использования:

- **«Вспомнить»** — поводы листать, видеть, что добавили друзья. Скрапбук-эстетика. Главный экран — `Доска`.
- **«Найти»** — инструмент: «куда сходить сегодня». Чистый список с фильтрами. Главный экран — вкладка `Найти`.

Обе модели сосуществуют. Доска не должна быть удобной поисковой системой, поиск не должен быть скрапбуком. Это разные эмоциональные состояния.

## Структура

5 нижних вкладок: `Доска · Карта · Найти · Я · ⊕`. Кнопка `⊕` — добавить (новая запись / визит / запись от руки / wishlist).

Альтернатива: 4 вкладки + кнопка `+ добавить` в шапке Доски (как в макете). Решить с разрабом по эргономике.

## Экраны

### Доска (главная)

См. макет `v2` в `index.html`.

**Поведение:**
- **Общая** на круг друзей — все добавляют, все видят. Без модерации.
- Автообновляется по мере добавления записей.
- Никаких удалений со стороны системы — всё уезжает в архив с подгрузкой по скроллу.
- Текущая неделя — 4-5 артефактов с воздухом + кнопка «↓ ещё N на этой неделе».
- Старые недели — свернутые «полоски» с датой, мини-стопкой полароидов и счётчиком (`27 апр – 3 мая · 6 мест · 1 ♦`). Тап «раскрыть ↓» — расширяет до полноценного скрапбук-разворота.
- Бесконечный скролл назад с подгрузкой по неделям.

**Артефакты на доске:**
- Полароид — `place_id` + фото + дата
- Билетик-рейтинг — `food/service/vibe`
- Кружочек видео — короткое видеосообщение (уже есть на бэке)
- Записка-note — текст от руки (новый тип контента)
- Штамп — кухня / город / тег
- Gem-знак — у жемчужин, с анимацией блика

**Каждый артефакт несёт автора** — маленькая аватарка-кружок в углу (`AuthorTag`), чтобы было видно, кто прикнопил.

**Wishlist на доске:** записка про «не забыть в Тбилиси» — это запись из общего wishlist. Когда жемчужина «найдена» (есть визит) — записка остаётся, но *перечёркивается красным маркером* и рядом появляется полароид визита. Эффект «исполнения желания».

**Кнопка «+ добавить»** — справа в линии заголовка текущей недели. По тапу — модалка с выбором типа: визит / запись от руки / wishlist / только фото. Дальше — соответствующая форма (см. ниже).

### Найти (поиск + навигация по разрезам)

**Контекстный, не глобальный.** Вход в скоуп — потом фильтры/поиск внутри скоупа.

**Структура страницы (сверху вниз):**

1. **Поисковая строка** — стилизованная под штамп: «найти место, кухню, друга». Ищет в скоупе (по умолчанию — все места). Если активен другой скоуп (например, «Анины места») — ищет в нём, под полем ссылка «искать везде».

2. **Кнопка «🎲 Мне повезёт»** — рандомное место. С учётом активных фильтров (если открыт скоуп «Тбилиси» — рандомное место в Тбилиси).

3. **Полки навигации** (горизонтальные карусели или вертикальные подзаголовки):
   - **Жемчужины** — превью топ-5, тап «все →» ведёт на hub `/gems`
   - **По городам** — список с количеством мест: `Москва · 87`, `СПб · 23`, `Тбилиси · 14`. Тап → `/cities/:name`
   - **По друзьям** — список друзей с количеством мест и жемчужин. Тап → `/people/:id`
   - **По кухням** — кухни с количеством

4. **Кнопка «фильтры»** — открывает шторку (см. ниже).

5. **Список результатов** — единый список карточек заведений (когда применены фильтры или в скоупе).

**Шторка фильтров (MVP):**
- Город (single)
- Кухня (multi — уже есть)
- Категория (multi — уже есть)
- Рейтинг от: для food / service / vibe (3 слайдера 0–10)
- Только жемчужины (toggle)
- Кто был (multi-select друзей) — *новое*
- Дата визита: от / до — *новое, опционально в MVP*
- Сортировка: рейтинг ↓, рейтинг ↑, новизна, алфавит, дата визита

**Сортировка по чьему рейтингу:** переключатель в шапке списка — `Средний / Анин / Мой`. По умолчанию — средний по кругу.

**Карточка результата (компактная):**
- Серифа-название места + город · кухня (мелко)
- Аватарки тех, кто был (стек справа)
- Билетик с агрегированным рейтингом
- Маленький gem-значок, если жемчужина

**Тап карточки** → `/places/:id` (страница места).

### Карта

Существующая страница `/map` (Яндекс.Карты). MVP-задача — рестайл маркеров.

- Маркеры под скрапбук — стилизация под кнопку-канцелярку (красная головка) или штампик.
- Жемчужины — отдельный маркер с gem-знаком + лёгким бликом.
- Кластеризация — оставить как есть.
- Balloon — серифа-имя, билетик-рейтинг, аватарки авторов.
- Из шапки — кнопка «фильтры» (общая со страницей «Найти», см. backend).

Из результатов поиска кнопка «↗ показать на карте» открывает карту с теми же фильтрами.

### Я (личный кабинет)

**Сверху:**
- Аватарка крупно (уже есть upload через imageutil)
- Имя + дата регистрации
- Личная стата: «32 места · 5 жемчужин · 4 города» (билетик-стиль)

**Подразделы (вкладки или секции):**
- **Мои визиты** — хронология моих отзывов (как лента, но только мои). Уже есть в `Profile.vue`, нужно перевести на скрапбук-вёрстку.
- **Мой wishlist** — что я хочу сходить. Уже есть на бэке (общий + custom). Стилизовать под записочки.
- **Заметки** — записки от руки, которые я оставлял (отдельный тип контента из доски, отфильтрованный по author = me).
- **Настройки** — смена пароля (есть), аватарка (есть), инвайты (есть).

**Кнопка «выход».**

### Место — `/places/:id`

Существующая `PlaceDetail.vue`. Перевести вёрстку на скрапбук, контент — тот же.

**Шапка:**
- Серифа-название (крупно)
- Город · кухня · категория (штампики)
- Gem-значок, если жемчужина (анимированный)
- Карта-превью с маркером

**Тело:**
- Галерея фото (полароидами, скотч)
- Кнопки: «оставить отзыв», «в wishlist», «↗ карта»
- Список отзывов — каждый как «разворот»: автор-аватарка, дата от руки, фото (полароид), билетик-рейтинг, текст, кружочек видео, кнопка «прикнопить на доску» (если автор != я).

### Профиль друга — `/people/:id` (новое)

Публичный профиль одного из круга.

- Аватарка крупно + имя + «в кругу с …»
- Стата: «32 места · 5 жемчужин · 4 города»
- Подзакладки:
  - **Все места** — её/его визиты в виде ленты
  - **Жемчужины** — только её жемчужины
  - **По городам** — группировка визитов по городам (мини-секции)

«Кнопка прикнопить на доску» рядом с любой записью.

Шарится ссылкой `/people/:id` — это инвайт-only приложение, но внутри круга — расшариваемо.

---

### Город — `/cities/:name` (новое)

Городская страница как путеводитель.

- Шапка: название города (большая серифа), стата `87 мест · 12 жемчужин · 4 человека добавляли`
- Большая иллюстрация-открытка (placeholder под фото) или мини-карта
- Подзакладки:
  - **Жемчужины** — все жемчужины в этом городе
  - **Все места** — список с фильтрами
  - **По кухням** — кухни в этом городе
- Шарится ссылкой `/cities/Tbilisi` — расшариваемая страница для своих.

---

### Жемчужины hub — `/gems` (новое)

Всё про жемчужины. Структура — как `Найти`, но пред-отфильтровано на `is_gem = true`.

- Полки: «По городам», «По друзьям», «По кухням» (количество жемчужин в каждой)
- Список (с переключателями сортировки)
- В шапке — общая стата круга: «47 жемчужин · 12 городов · 4 куратора»

## Флоу: добавить визит

Вход — кнопка `+ добавить` на Доске или из карточки места.

1. **Выбор типа** — модалка-веер: визит / только фото / записка / wishlist
2. **Если визит:**
   - Выбрать место — поиск + Yandex geosuggest (есть). Если нет — создать новое (форма уже есть в `PlaceForm.vue`).
   - Дата визита (по умолчанию — сегодня)
   - Кто был — multi-select из круга
   - Фото — несколько (новое: галерея вместо одного `image_url` — есть в roadmap)
   - Видеосообщение — опционально (есть в коде)
   - Рейтинги: food / service / vibe (0-10) — есть
   - Жемчужина? — toggle
   - Текст отзыва — есть
3. **После сохранения** — артефакт прикнопывается на Доску.

## Флоу: записка от руки

Вход — кнопка `+ добавить` → «записка».

- Просто текстовое поле, без привязки к месту (или с опциональной привязкой)
- Опционально: цвет бумаги, цвет скотча
- После сохранения — на доску в текущую неделю.
