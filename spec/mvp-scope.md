# AEVA Eat — MVP scope

## Что делаем в MVP

### Дизайн-система
- Полная скрапбук-CSS (`scrapbook.css` уже есть в прототипе)
- Шрифты Lora + Caveat
- Все примитивы: polaroid, tape, stamp, ticket, note, gem-badge, author-tag, kruzhok, pin-btn

### Главный экран
- Доска: текущая неделя (4-5 артефактов с воздухом + «↓ ещё N»)
- Свернутые полоски прошлых недель с раскрытием
- Бесконечный скролл назад
- Кнопка «+ добавить»
- Анимация gem-gleam

### Найти
- Поиск-строка (контекстный)
- Кнопка «🎲 Мне повезёт»
- Полки: Жемчужины / Города / Друзья / Кухни
- Шторка фильтров: город / кухня / категория / рейтинги / жемчужины / кто был / сортировки
- Список результатов

### Карта
- Существующая, рестайл маркеров под штампики
- Жемчужины — особый маркер с блеском
- Общие фильтры с «Найти»

### Я
- Профиль (есть) — перевёрстка на скрапбук
- Мои визиты (хронология)
- Мой wishlist (стилизация под записочки)
- Мои заметки (новое — отфильтровано по author = me)
- Настройки (есть)

### Новые страницы
- `/places/:id` — перевёрстка существующей под скрапбук + галерея фото + кнопка «прикнопить на доску»
- `/people/:id` — публичный профиль друга с подзакладками
- `/cities/:name` — городская страница с подзакладками
- `/gems` — hub жемчужин

### Бэк-дельта (см. `backend.md`)
- `notes` — записки
- `feed_events` — лента
- Эндпоинты `/feed`, `/cities`, `/users/:id`, `/gems`, `/random`
- Расширение `/places` фильтрами `attended_by`, `visit_from/to`, сортировками
- Несколько фото в review (`review_photos`)
- `wishlist/all`

### Wishlist → визит
- Триггер: при создании review автоматически помечать соответствующую wishlist-запись `struck=true`
- Визуально: записка зачёркивается, рядом полароид

### Видеосообщения
- Уже работают как часть review. Расширить — отдельный артефакт без визита (опционально в MVP).

## Что НЕ делаем в MVP

### Перенесли на next
- Тёмная тема
- PWA (опционально, если бюджет позволит)
- Транскод видео (ffmpeg)
- Админка / dashboard для superuser
- pgAdmin
- Часы работы заведения, средний чек, статус (открыто/закрыто)
- Теги от пользователей (веранда / тихо / детям ок) — НЕ в MVP, в next
- Комментарии к отзывам
- Лента активности друзей как уведомления
- Персональные «итоги года / итоги месяца» — отложили, обсудим позже
- Множественное число городов в фильтре (single only)
- CI/CD, бэкап БД на S3

### Решено убрать
- Жемчужина месяца — нет такого режима
- Группировка ленты по месяцам — нет, скроллим неделями

## Открытые вопросы (нужны решения от тебя)

1. **5-я вкладка `⊕ Добавить`** в нижнем тулбаре — да или достаточно кнопки в шапке Доски?
2. **Дата визита в фильтрах** — MVP или next?
3. **Сортировка по рейтингу конкретного человека** — это переключатель в шапке списка. Делаем в MVP или хватит «средний по кругу»?
4. **Видеосообщение без визита** — отдельный тип артефакта в MVP, или только как часть review?
5. **Записки могут быть без привязки к месту** — да? (предполагаю да)
6. **Ширина круга** — сколько человек ожидаем (3-5? до 30?). Это влияет на UI: например, multi-select «кто был» из 5 человек может быть кружочками-чекбоксами; из 30 — нужен list/search.
