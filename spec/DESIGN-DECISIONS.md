## Лента (Доска)

### L1 · Группировка отзывов одного дня — Вариант A

Один полароид на place, авторы стекаются в `AuthorTag`. Лента — это «что у нас было», а не «кто что написал». Если три человека за один вечер сидели в Probka — это одно событие, а не три.

Внутри страницы места — там уже да, отдельные отзывы.

**Edge case:** один и тот же человек пришёл в место в разные дни — это разные артефакты. Группируем по `(place_id, date, attendees)`, не по `(place_id, date)` глобально.

> Реализация edge case требует `feed_events` (round 4 backend). Сейчас feed группируется только по place — для MVP приемлемо.

### L2 · Cover fallback — ✅ оправдан

Без фото скрапбук разваливается. Приоритет:
1. `place.image_url`
2. Первое review-фото
3. Пастельный плейсхолдер (через хеш `place.id` чтобы цвет был стабильным)

Уже сделано на бэке, оставляем.

### L3 · Несколько фото в review — стопка полароидов

Это и есть скрапбук. Не «+ ещё 2» — это интерфейс ленты Telegram, не дневника.

| Фото | Раскладка |
|---|---|
| 1 | один полароид (как сейчас) |
| 2 | два внахлёст, второй чуть выезжает справа, наклон −2° / +3° |
| 3+ | стопка из 3-х (показываем 3, в углу маленькая рукописная пометка `+ еще 2` каракулями, не круглый бейдж) |

На детали места — все фото в линию, скроллится горизонтально.

> Ждёт `review_photos` таблицу на бэке (round 4).

### L4 · Развёрнутая прошлая неделя

Тот же 2-колоночный масонри. Главное — не вводить новый ритм при разворачивании.

**Микро:** у развёрнутой архивной недели приглушить контраст — бумага чуть темнее (`--paper-deep` вместо `--paper`). На 5–7% разницы хватит.

## Профиль и аватары

### P1 · Размеры AuthorTag

| Пресет | Размер | Где |
|---|---|---|
| `lg` | 28px | Шапка страницы места («с нами были…») |
| (default) | 22px | Лента, ResultCard, ReviewCard |
| `sm` | 18px | Очень плотные места (архивная свернутая полоска) |

22px — дефолт, не трогаем.

### P2 · Цветной ободок при фото — НЕТ

Фото — это уже идентификация. Цветной ободок поверх фото = шумно и колхозно. Цветовая привязка работает только когда инициал.

При фото — нейтральный белый ободок 2px (как у полароидов) + лёгкая тень.

### P3 · Стата-билетик в профиле

Оставить **мест / жемчужин / городов**. Это путеводитель, а не профиль соцсети.

«Число фотографий» / «число отзывов» — метрики тщеславия, не подходят к камерному дневнику.

**Опционально (next):** четвёртая строка — «любимая кухня» рукописным под билетиком: «грузинская — 11 раз». Это уже не цифра, а характер.

## Карта

### M1 · Маркеры с рейтингом и светофором

Цветовая логика:
- ≥8 → moss (зелёный кайф)
- ≥5 → ochre
- <5 → terra (красный нет)
- **Без рейтинга / wishlist → ochre** (не terra! terra = «плохо», нельзя ей же красить «не оценили». Иначе wishlist = помойка)
- **Цифра рейтинга — только если есть отзыв.** Без отзыва — пустая головка кнопки. Читается как «здесь нас ещё не было»

Жемчужины — ромбик в охре с halo, рейтинг внутри.

### M2 · Balloon

Добавить **стек авторов** под рейтингом, перед ссылкой «подробнее →». 3-4 аватарки (P1 default 22px).

Без слова «с нами:», просто двоеточие или ничего.

> Голос: двоеточие без слова, для лаконичности.

## Запись отзыва

### R1 · Photo dropzone

Один пустой полароид-плейсхолдер с пунктиром. После загрузки — он становится полароидом фото, рядом справа появляется второй пустой плейсхолдер с лёгким наклоном (как «следующий лист в стопке»).

**Удаление:** крестик в углу полароида.
**Лимит:** 5 фото. После 5-го — следующий пустой не появляется.

> Сейчас, пока бэк не готов: один слот — ОК, можно отдавать. Полная реализация после `review_photos`.

### R2 · Видеозапись через WiFi — known limitation

Не поднимать HTTPS-туннель ради круга друзей. Запись видео — фича-роскошь.

**UI:** вместо «требует https» — мягче, в духе скрапбука: **«кружочки пока только с компьютера»** рукописно. Технические причины оставить в `console.error` для разраба.

### R3 · gem-toggle — ✅

Pill с пунктирной рамкой → анимированный gem при включении.

**Микро:** при клике короткая анимация «штампа» (быстрое масштабирование 1.0 → 0.85 → 1.05 → 1.0 за 200мс). «Прихлопнул печатью».

## Найти

### F1 · Полки — добавить ритма

Сейчас три горизонтальные полки подряд — однообразно. Каждая полка получает свой тип размещения:

| Полка | Раскладка |
|---|---|
| Жемчужины | крупная горизонтальная карусель |
| **Города** | **вертикальный список, имя слева серифой, count-штамп справа** |
| Кухни | горизонтальные moss-штампы в одну линию с переносом (wrap) |
| Друзья (когда будет) | горизонтальная карусель аватарок 60px с подписью именем и count |

Три разных типа размещения подряд — это и есть скрапбук-ритм.

### F2 · 🎲 «Мне повезёт»

Клиентский MVP достаточен. `exclude_visited_by=me` — приятная роскошь, без неё «повезёт» работает: выпадает место, ты его уже видел — окей, жмёшь ещё раз. Серверный — в next.

## Глобальные

### G1 · Mobile-first vs адаптив

ОК, оставляем. Скрапбук по природе — узкий формат. Полароиды в три колонки на десктопе — это уже Pinterest.

Узкая колонка центром (макс. 480px для контентных, 600px для Найти) на широких экранах — правильно.

**Микро:** фон вокруг колонки на десктопе — не белый/паперный, а чуть темнее (`--paper-edge`). Будто «лист бумаги на столе». Одна строка CSS, сильно повышает воспринимаемое качество на десктопе.

### G2 · Тёмная тема — next, не MVP

Скрапбук плохо переводится в тёмную: бумага = свет, чернила = тень. Тёмная требует отдельной концепции (доска вместо бумаги? пробковый щит ночью?).

Не торопимся, делаем осознанно потом.

### G3 · 5-я вкладка `+ Добавить` — НЕТ

4 таба + кнопка в шапке Доски — правильно. 5-я вкладка `+` — это паттерн соцсетей, где добавление контента — главное действие. У нас главное — смотреть и вспоминать.

Альтернатива на потом: кнопка `+` в шапке любого экрана рядом с wordmark `aeva·eat`. Не в табваре. Но это next.
